# Token Budget Validation Checklist

## Skill Files (Tier 1: ≤15KB)

- [ ] All SKILL.md files under 15KB
- [ ] No oversized skill content
- [ ] Skills delegate to sub-skills when large
- [ ] Token-heavy content in separate docs

## Agent Files (Tier 2: ≤20KB)

- [ ] All agent files under 20KB
- [ ] Prompts concise and focused
- [ ] No duplicated instructions
- [ ] Templates used for repetitive content

## Command Files (Tier 3: ≤10KB)

- [ ] All command files under 10KB
- [ ] Commands focused on single purpose
- [ ] No embedded documentation (link instead)
- [ ] Examples concise

## Documentation (Tier 4: ≤25KB)

- [ ] Core docs under 25KB each
- [ ] Long docs split into sections
- [ ] Tables used over prose
- [ ] Examples minimal but clear

## Checklists (Tier 5: ≤2KB)

- [ ] Checklists under 2KB each
- [ ] Focused on verification items
- [ ] No verbose explanations
- [ ] Links to full docs

## Context Files (Tier 6: ≤5KB)

- [ ] CLAUDE.md under limits
- [ ] Session context under limits
- [ ] Recent changes trimmed

## Templates (Tier 7: ≤3KB)

- [ ] Template files minimal
- [ ] Placeholders clear
- [ ] No embedded documentation

## Total Budget (Tier 8: ≤500KB)

- [ ] Total .claude/ directory under 500KB
- [ ] Growth monitored
- [ ] Deprecated content removed
- [ ] Archives cleaned

## Optimization

- [ ] Redundant content eliminated
- [ ] Tables over lists over prose
- [ ] External links vs duplication
- [ ] Caching strategies applied
