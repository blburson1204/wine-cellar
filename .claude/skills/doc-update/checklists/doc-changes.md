# Documentation Change Categorization Checklist

## Foundational Changes (REQUIRE Documentation)

- [ ] New architectural patterns introduced
- [ ] New concepts added to system
- [ ] Behavior changes affecting multiple components
- [ ] Breaking changes to APIs
- [ ] New development workflows

## Clarifications (OPTIONAL Documentation)

- [ ] Fixing outdated information
- [ ] Improving existing explanations
- [ ] Adding missing examples
- [ ] Correcting errors

## Implementation Details (AVOID Documentation)

- [ ] How specific code works internally
- [ ] Internal function implementations
- [ ] Private APIs
- [ ] Temporary solutions

## Over-Documentation Signals

- [ ] Documenting obvious code
- [ ] Duplicating code comments
- [ ] Explaining standard patterns
- [ ] Documenting implementation details

## Documentation Targets

- [ ] CLAUDE.md for patterns/workflows
- [ ] README for setup/getting started
- [ ] Architecture docs for system design
- [ ] API docs for public interfaces
- [ ] Migration guides for breaking changes

## Quality Criteria

- [ ] Documentation concise
- [ ] Examples functional
- [ ] Links not broken
- [ ] Versioning clear
- [ ] Audience identified

## Maintenance

- [ ] Outdated docs removed
- [ ] Deprecated content marked
- [ ] Versions documented
- [ ] Changelog updated
